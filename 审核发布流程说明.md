# 提交审核发布全流程说明

## 1. 状态定义

所有内容（景点、美食、文化、攻略、酒店、体验项目等）都有统一的状态字段：

- **0：待审核** - 用户刚刚提交的内容，等待管理员审核
- **1：已通过** - 审核通过，内容可正常展示给用户
- **2：已拒绝** - 审核拒绝，内容不会展示给用户

---

## 2. 用户提交流程

### 2.1 普通用户提交内容

普通用户通过以下接口提交内容：

- 攻略提交：`POST /strategies/submit`
- 景点提交：`POST /attractions/submit`
- 美食提交：`POST /foods/submit`
- 文化提交：`POST /cultures/submit`
- 酒店提交：`POST /hotels/submit`
- 体验项目提交：`POST /experiences/submit`

### 2.2 提交流程代码示例（以攻略为例）

```java
// StrategyController.java - submitStrategy方法
@PostMapping("/submit")
@PreAuthorize("hasAnyRole('USER', 'ADMIN')")
public Result<Strategy> submitStrategy(@RequestBody Strategy strategy, HttpServletRequest request) {
    // 1. 从Token中获取用户ID
    String token = getTokenFromRequest(request);
    Long userId = jwtUtil.getUserIdFromToken(token);
    
    // 2. 查询用户信息，判断角色
    User user = userMapper.selectById(userId);
    
    // 3. 设置用户ID
    strategy.setUserId(userId);
    
    // 4. 根据角色设置状态：管理员直接通过(1)，普通用户待审核(0)
    boolean isAdmin = RoleConstants.isAdmin(user.getRole());
    strategy.setStatus(isAdmin ? 1 : 0);
    
    // 5. 初始化统计字段
    strategy.setViewCount(0);
    strategy.setCommentCount(0);
    strategy.setFavoriteCount(0);
    strategy.setLikeCount(0);
    strategy.setIsRecommend(0);
    
    // 6. 处理分类等字段
    // ...
    
    // 7. 保存到数据库
    strategyMapper.insert(strategy);
    
    // 8. 根据角色返回不同提示
    String message = isAdmin ? "攻略保存成功，已直接发布" : "攻略提交成功，等待审核";
    return Result.success(message, strategy);
}
```

### 2.3 提交时的关键处理

1. **权限控制**：只有登录用户（USER或ADMIN）才能提交内容
2. **状态设置**：
   - 普通用户提交：status = 0（待审核）
   - 管理员提交：status = 1（已通过）
3. **字段初始化**：浏览数、评论数、收藏数、点赞数初始化为0
4. **字段验证**：对字段进行验证（如bestSeason限制长度为50字符）

## 3. 管理员审核流程

### 3.1 待审核内容查询

管理员通过以下接口查询待审核内容：

- 待审核景点：`GET /admin/pending/attractions`
- 待审核美食：`GET /admin/pending/foods`
- 待审核文化：`GET /admin/pending/cultures`
- 待审核攻略：`GET /admin/pending/strategies`
- 待审核酒店：`GET /admin/pending/hotels`
- 待审核体验项目：`GET /admin/pending/experiences`

查询条件：status = 0，按创建时间倒序排列。

### 3.2 审核操作接口

管理员通过以下接口审核内容：

- 审核景点：`PUT /admin/attractions/{id}/audit`
- 审核美食：`PUT /admin/foods/{id}/audit`
- 审核文化：`PUT /admin/cultures/{id}/audit`
- 审核攻略：`PUT /admin/strategies/{id}/audit`
- 审核酒店：`PUT /admin/hotels/{id}/audit`
- 审核体验项目：`PUT /admin/experiences/{id}/audit`

参数：status（1=通过，2=拒绝）

### 3.3 审核流程代码示例（以景点为例）

```java
// AdminController.java - auditAttraction方法
@PutMapping("/attractions/{id}/audit")
@PreAuthorize("hasRole('ADMIN')")
public Result<?> auditAttraction(@PathVariable("id") Long id, @RequestParam("status") Integer status) {
    // 1. 检查内容是否存在
    Attraction attraction = attractionMapper.selectById(id);
    if (attraction == null) {
        return Result.notFound("景点不存在");
    }
    
    // 2. 验证状态参数合法性
    if (status != 1 && status != 2) {
        return Result.error("状态参数错误，1-通过，2-拒绝");
    }
    
    // 3. 如果审核通过，且地址有值但经纬度为空，尝试自动获取经纬度
    if (status == 1 && attraction.getAddress() != null && !attraction.getAddress().trim().isEmpty()) {
        if (attraction.getLongitude() == null || attraction.getLatitude() == null) {
            System.out.println("景点审核通过 - 尝试自动获取经纬度，地址: " + attraction.getAddress());
            Map<String, BigDecimal> location = mapService.geocodeAddress(attraction.getAddress(), "太原");
            if (location != null && location.containsKey("longitude") && location.containsKey("latitude")) {
                attraction.setLongitude(location.get("longitude"));
                attraction.setLatitude(location.get("latitude"));
                System.out.println("景点审核通过 - 成功获取经纬度: " + attraction.getLongitude() + ", " + attraction.getLatitude());
            }
        }
    }
    
    // 4. 更新状态
    attraction.setStatus(status);
    attractionMapper.updateById(attraction);
    
    // 5. 返回结果
    return Result.success(status == 1 ? "审核通过" : "审核拒绝");
}
```

### 3.4 审核时的关键处理

1. **权限控制**：只有ADMIN角色才能执行审核操作
2. **参数验证**：status只能是1或2
3. **自动补全经纬度**：审核通过时，如果地址有值但经纬度为空，自动调用高德地图API获取经纬度
4. **状态更新**：更新内容的status字段

### 3.5 美食审核的特殊处理

美食审核时，会处理推荐餐厅的经纬度：

```java
// 如果审核通过，且推荐餐厅中有地址但缺少经纬度，尝试自动获取经纬度
if (status == 1 && food.getRecommendedRestaurants() != null && !food.getRecommendedRestaurants().isEmpty()) {
    try {
        JSONArray restaurants = JSON.parseArray(food.getRecommendedRestaurants());
        if (restaurants != null && restaurants.size() > 0) {
            boolean needUpdate = false;
            for (int i = 0; i < restaurants.size(); i++) {
                JSONObject restaurant = restaurants.getJSONObject(i);
                String address = restaurant.getString("address");
                if (address != null && !address.trim().isEmpty()) {
                    // 检查是否缺少经纬度
                    if (!restaurant.containsKey("longitude") || !restaurant.containsKey("latitude") ||
                        restaurant.get("longitude") == null || restaurant.get("latitude") == null) {
                        System.out.println("美食审核通过 - 尝试自动获取餐厅经纬度，地址: " + address);
                        Map<String, BigDecimal> location = mapService.geocodeAddress(address, "太原");
                        if (location != null && location.containsKey("longitude") && location.containsKey("latitude")) {
                            restaurant.put("longitude", location.get("longitude"));
                            restaurant.put("latitude", location.get("latitude"));
                            needUpdate = true;
                        }
                    }
                }
            }
            if (needUpdate) {
                food.setRecommendedRestaurants(restaurants.toJSONString());
            }
        }
    } catch (Exception e) {
        // 异常处理
    }
}
```

## 4. 发布展示流程

### 4.1 内容展示过滤

前端展示内容时，只查询status=1的内容：

```java
// StrategyServiceImpl.java - getStrategyList方法
@Override
public PageVO<Strategy> getStrategyList(String category, String theme, Boolean isRecommend, Integer page, Integer size) {
    Page<Strategy> pageParam = PageUtil.createPage(page, size);
    LambdaQueryWrapper<Strategy> wrapper = new LambdaQueryWrapper<>();
    
    if (category != null && !category.isEmpty()) {
        wrapper.eq(Strategy::getCategory, category);
    }
    if (theme != null && !theme.isEmpty()) {
        wrapper.eq(Strategy::getTheme, theme);
    }
    if (isRecommend != null && isRecommend) {
        wrapper.eq(Strategy::getIsRecommend, 1);
    }
    
    // 只查询已通过审核的内容
    wrapper.eq(Strategy::getStatus, 1);
    
    wrapper.orderByDesc(Strategy::getIsRecommend);
    wrapper.orderByDesc(Strategy::getViewCount);
    wrapper.orderByDesc(Strategy::getCreateTime);
    
    Page<Strategy> result = strategyMapper.selectPage(pageParam, wrapper);
    return new PageVO<>(result.getTotal(), result.getPages(), result.getCurrent(), result.getSize(), result.getRecords());
}
```

### 4.2 详情页展示

查看详情时，也会验证状态：

```java
// StrategyServiceImpl.java - getStrategyDetail方法
@Override
@Transactional(rollbackFor = Exception.class)
public Strategy getStrategyDetail(Long id) {
    Strategy strategy = strategyMapper.selectById(id);
    // 验证内容是否存在且已通过审核
    if (strategy == null || strategy.getStatus() != 1) {
        throw new BusinessException("攻略不存在");
    }
    
    // 增加浏览次数
    strategy.setViewCount((strategy.getViewCount() == null ? 0 : strategy.getViewCount()) + 1);
    strategyMapper.updateById(strategy);
    
    return strategy;
}
```

## 7. 审核流程亮点

1. **角色差异化处理**：管理员提交直接通过，普通用户提交需审核
2. **自动补全经纬度**：审核通过时自动调用高德地图API补全经纬度
3. **统一状态管理**：所有内容使用统一的status字段
4. **完整的操作日志**：所有审核操作都会记录操作日志
5. **地图智能集成**：地址转经纬度、范围过滤等功能
